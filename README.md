# 순수 HTML5 3D 카드
순수한 HTML5와 CSS를 사용하여 3D 카드를 구현했습니다.

## **기능**
* **카드**: 클릭 후 드래그하여 카드를 움직일 수 있습니다.

## **동작 원리**

### 박스 조립

모든 면은 `top: 50%; left: 50%`로 일단 박스 중심에 모은 뒤, 음수 `margin`으로 자기 크기의 절반만큼 되돌려 정중앙에 정렬됩니다.
이 상태에서 `translateZ` 하나만으로 각 면을 바깥으로 밀어내면 좌표 계산의 실수 없이 박스가 닫힙니다.

- 앞/뒤 면: `translateZ(+-d/2)` (단, `d`는 카드 두께)
- 좌/우 면: `rotateY(+-90deg) translateZ(w/2)`
- 상/하 면: `rotateX(+-90deg) translateZ(h/2)`

`backface-visibility: hidden` 으로 뒤를 향한 면을 미리 컬링해 불필요한 합성을 줄일 수 있습니다.

### 둥근 모서리 보정

앞/뒤 면에는 `border-radius`가 걸려 모서리가 안쪽으로 `r`만큼 휘어드는데, 이 때문에 두께를 채우는 옆면 strip 을 면 크기 그대로 두면 둥근 모서리 밖으로 삐져나와 이를 막기 위해 옆면 strip의 길이를 양쪽 모서리 반지름만큼(`치수 - 2r`) 줄여 strip이 정확히 직선 구간만 덮도록 했습니다

하지만 이 방식은 사각 탭이 튀어나오는 문제를 없애주었지만 곡면 자체를 만들어내지는 못한다는 점이 있습니다.

### 프레임레이트 독립 관성

핵심은 회전 속도를 "프레임당 각도"가 아니라 **"밀리초당 각도(deg/ms)"** 로 다룬다는 점인데

1. **드래그 중**: 포인터 이동량을 직전 이벤트와의 시간 간격 `dt`로 나눠 `deg/ms` 단위 속도를 기록하게 됩니다. 천천히 끌든 빠르게 긋든, 이렇게 측정된 속도는 입력 장치의 보고 주기에 좌우되지 않습니다
2. **놓은 뒤 - coast**: 매 프레임 경과 시간 `dt`에 대해 감속을 적용하는데, 단순히 속도에 상수를 곱하는 대신 지수 감속의 적분으로 이동량을 구합니다

   속도가 `v * e^(-DECAY * t)`로 줄어들 때, `dt` 구간 동안 실제로 회전하는 총량은

   ```
   회전량 = v * (1 - e^(-DECAY * dt)) / DECAY
   ```

   입니다. 코드에서는 `k = Math.exp(-DECAY*dt)` 를 구해 `rot += vel * (1-k) / DECAY` 로 적용하고, 속도 자체는 `vel *= k` 로 줄였습니다.
   프레임을 촘촘히 그리든 띄엄띄엄 그리든 구한 적분값은 변하지 않으므로 총 회전 궤적이 주사율과 무관하게 동일합니다.

4. **예외 처리**: 한 프레임의 `dt`를 `MAX_DT(64ms)` 로 클램프해, 탭을 백그라운드로 보냈다 돌아왔을 때 누적된 큰 시간 간격이 카드를 한 번에 날려보내는 것을 막았습니다.
   또한 드래그를 멈춘 채 일정 시간 이상 들고 있다 떼면 남아있던 오래된 속도를 버려 의도치 않은 관성이 발생하지 않게 해봤습니다.

### 스크린 고정 광택

* 광원은 화면 우상단 앞쪽에 고정된 방향광으로 구성했고 카드가 돌아도 빛은 화면에 붙어 있어서 카드를 기울이면 하이라이트가 표면 위를 미끄러지게 표현했습니다
* 하이라이트 위치 = dot(H, right), dot(H, up) 으로 면 위 좌표를 구해 CSS 변수(--gx/--gy) 로 넘겼습니다.
* 오버레이는 mix-blend-mode: screen 이라 이미지를 덮어쓰지 않고 빛을 더하는 방식으로 그라디언트 stop은 물리 공식이 아니라 부드러운 falloff 를 위해 직접 튜닝했습니다.
